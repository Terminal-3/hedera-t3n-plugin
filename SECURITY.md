# Security Policy

## Supported versions

Security fixes are currently targeted at the latest `2.x` release line.

## Reporting a vulnerability

Please do **not** report security issues in public GitHub issues or pull requests.

Preferred reporting path:

1. Open a private report through GitHub Security Advisories for this repository.
2. If that is unavailable, contact the repository maintainers through a private GitHub channel and include repro steps, impact, and any suggested mitigation.

## What to include

Please include:

- affected version(s)
- impact and attack scenario
- reproduction steps or proof of concept
- any known mitigations or workarounds

## Response expectations

- initial acknowledgment target: within 72 hours
- remediation target: within 30 days for confirmed issues, depending on severity and dependency constraints

## Scope

This policy covers vulnerabilities in this plugin repository. Issues that belong primarily to upstream dependencies or external services should be reported to the relevant maintainers as well.
